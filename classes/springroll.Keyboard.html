detectNextKey

(
  • callback
  • [cancelKey="esc"]
  • [allowModifiers=false]
)

Defined in detectNextKey:273

Begins listening for the next key to be pressed (or repeated). The callback is passed the name of that key, so that it can then be used with other functions. Additionally, if the key pressed does not have a recognized keyCode, then an entry is created for it with the name 'key_%keyCode%'. If the cancel key is pressed, then the callback is passed null. If allowModifiers is true, then the result should be considered a key combination to be used with addCombo().

Parameters:

  • callback Function

    The function to be called when a key is pressed.

  • [cancelKey="esc"] String optional

    The name of the key to cancel the listener. If explictly passed null, then the "esc" key can be detected and this action can only be cancelled with stopDetecting().

  • [allowModifiers=false] Boolean optional

    If key modifiers (ctrl, alt, shift, command) should be detected as modifications of the pressed key instead of as their own keypresses.
